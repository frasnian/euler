If we list all the natural numbers below 10 that are multiples of 3 or 5, we get 3, 5, 6 and 9. The sum of these multiples is 23.
Find the sum of all the multiples of 3 or 5 below 1000.
There really isn't much to say about this solution - for numbers this small I just used a brute force approach (for idx = 3 to 999, if idx is evenly divisible by 3 or 5, add idx to running sum). We start at 3 because anything less is obviously not divisible by three.

The zero-runtime version does exactly the same thing, just using C++ template metaprogramming: we recursively instantiate a template that does the check for divisibility, and terminate the recursion with an explicit specialization that doesn't recurse.

About the only thing even mildly interesting about it is that you may run into template recursion limits for your compiler (see the readme file for compiler details).
