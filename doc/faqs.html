Euler Zero - FAQs

Okay, I haven't really gotten "frequent" questions from random github people yet, but I've definitely gotten a few common ones from friends:

Q) Isn't it wrong to post the solutions?
A) See this project wiki page.

Q) What's the point?
A) There isn't one. None. Why do you do crossword puzzles or Sudoku on the commuter train on your way to work?

Q) You've only got a few problems up so far. Why spend all this time on ancillary stuff like documentation and structure?
A) Well, there are a lot of problems in Project Euler so I'd rather plan ahead. If I keep adding more solutions, I never want to go back and re-do anything for something that is anything but a serious project. Spending the time now, when there are only a few solutions up, means that any issues that come up now occur when the entire project is still manageable and that I have to spend almost no time on this kind of stuff later. Also, if you know me you know that I'm a bit OCD when it comes to project organization. That's just how I roll.

Q) Do you actually find this fun, or is there something wrong with you?
A) Yes.
